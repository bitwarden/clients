## Display

Breadcrumbs are typically paired with page content to help users understand where they are within a
site or application and navigate back to parent pages. The current page should not appear as a
breadcrumb link.

### Sizes

Size should be chosen based on available space. Use the base size on large screen layouts where
there is ample horizontal space. Use the sm size in space-constrained contexts, such as smaller
screen widths and the extension.

<Canvas of={stories.Sizes} />

### Top Level

When a user is 1 level deep into a tree, the top level is displayed as plain text

<Canvas of={stories.TopLevel} />

### Second Level

When a user is 2 or more levels deep into a tree, the top level is displayed followed by an

<i class="bwi bwi-angle-right"></i> icon, and the following pages.

<Canvas of={stories.SecondLevel} />

### Overflow

When a user is more than 4 levels deep into a tree or if there is not enough available space, the
top level is displayed followed by an

<i class="bwi bwi-ellipsis-h"> </i> icon button that contains an overflow menu of breadcrumbs
exceeding the display limit/available space.

<Canvas of={stories.Overflow} />

## Small screens

If a screen's width is not large enough to display the full breadcrumb path, display a link to the
previous page and an <i class="bwi bwi-angle-right"></i> icon to take the user back to the previous
page.
