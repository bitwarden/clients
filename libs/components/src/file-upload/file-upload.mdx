## Variants

### Default

The default variant renders a styled file-input button alongside a filename readout. Use this for
single-file selection where a minimal footprint is preferred.

```html
<bit-file-upload accept=".png,.jpg,.gif,.svg" formControlName="file">
  <bit-label>Upload file</bit-label>
  <bit-hint>SVG, PNG, JPG or GIF (MAX. 800x400px)</bit-hint>
</bit-file-upload>
```

### Dropzone

The dropzone variant renders a drag-and-drop area. Users can either click the area to open the file
picker or drag files directly onto it. Add the `dropzone` attribute to opt in.

```html
<bit-file-upload dropzone [maxFileSize]="30" formControlName="file">
  <bit-label>Upload file</bit-label>
  <bit-hint>SVG, PNG, JPG or GIF (MAX. 800x400px)</bit-hint>
</bit-file-upload>
```

<Canvas of={stories.Dropzone} />

### Multiple files

Setting `multiple` automatically switches to the dropzone variant — the inline picker cannot accept
multiple files. A file list with name, size, and a delete button renders below the dropzone.

```html
<bit-file-upload [maxFileSize]="30" multiple formControlName="files">
  <bit-label>Upload files</bit-label>
  <bit-hint>SVG, PNG, JPG or GIF (MAX. 800x400px)</bit-hint>
</bit-file-upload>
```

The form value is always `File[]` (empty array means no file). Type the `FormControl` accordingly:

```ts
form = new FormGroup({
  files: new FormControl<File[]>([], { nonNullable: true }),
});
```

### Long filenames

Filenames that exceed the available width are truncated in the middle so both the start and file
extension remain visible. The first ~15% and last ~20% of the name are always shown, while the
middle ~65% is clipped with an ellipsis.

## Guidelines

### Choosing a variant

Use the **default** (button) variant for single-file fields where space is limited, such as an
avatar upload or an inline attachment field.

Use the **dropzone** variant when:

- The user may select multiple files (`multiple`)
- You want to encourage drag-and-drop as a primary interaction
- The field deserves visual prominence on the page

### `accept` — file type hint

Pass a comma-separated list of MIME types and/or file extensions to guide the browser's file picker.

| Example value            | Allows                 |
| ------------------------ | ---------------------- |
| `"image/*"`              | Any image              |
| `".pdf,.doc,.docx"`      | PDF and Word documents |
| `"audio/mpeg,audio/wav"` | Specific audio formats |
| `"image/*,.pdf"`         | Images and PDFs        |

> **Important:** `accept` is a browser hint only — it filters the file picker but does not prevent
> users from selecting other file types. Always validate file types on the server before processing.

See
[MDN: accept attribute](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Attributes/accept)
for a full list of valid specifiers.

### `maxFileSize` — size display hint

`maxFileSize` (in MB) is shown to the user inside the dropzone as "Max. File Size: X MB". It is
**not** enforced client-side. Validate file sizes in your component or on the server before
submitting the form.

### Binding selected files

Bind via Angular reactive forms. The form value is always `File[]` (empty array means no file
selected); single-file consumers read `value[0]`.

```html
<bit-file-upload dropzone [maxFileSize]="30" formControlName="file">
  <bit-label>Upload file</bit-label>
</bit-file-upload>
```

```ts
form = new FormGroup({
  file: new FormControl<File[]>([], { nonNullable: true }),
});

onSubmit() {
  const file = this.form.controls.file.value[0];
  // ...
}
```

You can pre-populate the list by calling `setValue([...])` on the control before the component
renders.

### Error state

The error border and message render automatically when the bound `FormControl` is `invalid` and
`touched`. Use Angular validators (or `setErrors`) on the control to drive the state.

```ts
form = new FormGroup({
  file: new FormControl<File[]>([], {
    nonNullable: true,
    validators: [
      Validators.required,
      (control) => {
        const file = control.value[0];
        return file && file.size > 30 * 1024 * 1024
          ? { tooLarge: { message: "File exceeds the 30 MB limit." } }
          : null;
      },
    ],
  }),
});
```

The component reads the first error from `control.errors` and renders it through `<bit-error>`,
matching the rest of the Bitwarden form-field family.

## Accessibility

- The default variant announces the selected filename via `aria-live="polite"` so screen readers
  notify the user after selection without interrupting the current reading position.
- Always provide a meaningful `<bit-label>` — it serves as the accessible label for the hidden file
  input inside both variants.
- Delete buttons in the file list are icon-only. The component supplies accessible labels through
  the i18n system (`delete` key), so no extra markup is needed.
