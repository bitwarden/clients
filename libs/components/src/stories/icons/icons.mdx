# Iconography

Bitwarden has a suite of presentational icons available for use, as well as a few helper classes for
sizing and rotation.

Icons use the `bwi` class, as well as an additional `bwi-*` class to indicate which icon will be
rendered.

Example basic usage:

```
<i class="bwi bwi-check"></i>
```

## Accessibility

Avoid using icons to convey information unless paired with meaningful, clear text. If an icon must
be used and text cannot be displayed visually along with the icon, use an `aria-label` to provide
the text to screen readers, and a `title` attribute to provide the text visually through a tool tip.
Note: this pattern should only be followed for very common iconography such as a settings cog icon
or an options menu icon.

# Developer Guide

## Adding a New Icon

Follow these steps to add a new icon to the icon font:

### 1. Export from Figma

1. Open the Figma icon file
2. Select the icon you want to export
3. Export as SVG with these settings:
   - **Outline stroke**: Enabled
   - **Include "id" attribute**: Disabled
   - **Simplify stroke**: Enabled
4. Name the file descriptively (e.g., `help.svg`, `star-filled.svg`, `add-circle.svg`)
   - **The filename becomes the icon name** - no mapping required!
   - `help.svg` → `bwi-help`
   - `star-filled.svg` → `bwi-star-filled`
5. Save to `libs/assets/src/material-icons/`

### 2. Generate the Icon Font

Run the build script:

```bash
npm run icons:build
```

This script will:

1. ✅ Create temporary variant copies (if needed for one-to-many mappings)
2. ✅ Generate the icon font files (woff2, woff, ttf, svg) using Figma names
3. ✅ Update the SCSS with the icon mappings
4. ✅ **Automatically update** `libs/components/src/shared/icon.ts` with the TypeScript icon list
5. ✅ Clean up temporary files

**Note**: Icons keep their Figma names throughout the process - no renaming or mapping needed! The
TypeScript icon array is now auto-generated, so you don't need to manually edit `icon.ts`.

### 3. Add to Storybook Documentation

Add your icon to the appropriate category in `libs/components/src/stories/icons/icon-data.ts`:

```typescript
// Choose the appropriate category:
// - statusIndicators
// - bitwardenObjects
// - actions
// - directionalMenuIndicators
// - miscObjects
// - platformsAndLogos

const actions = [
  // ... existing icons ...
  {
    id: "bwi-your-new-icon",
    usage: "Detailed description of when and how to use this icon.",
  },
];
```

**Usage description guidelines:**

- Start with what the icon indicates or does
- Include context-specific guidance (mobile, desktop, toggle states, etc.)
- Mention any variants or related icons
- Note any accessibility considerations

### 4. Test Your Icon

**Visual verification**: Run Storybook to see your icon

```bash
npm run storybook
```

Navigate to "Documentation / Icons" to verify your icon appears correctly

**Type checking**: Ensure TypeScript compiles without errors

```bash
npm run test:types
```

**Usage test**: Try using the icon in a component

```typescript
<bit-icon icon="bwi-your-new-icon"></bit-icon>
```

## Icon Naming Conventions

Icons use descriptive Figma names directly - the SVG filename becomes the icon class name:

- `help.svg` → `bwi-help`
- `star-filled.svg` → `bwi-star-filled`
- `add-circle.svg` → `bwi-add-circle`
- `arrow-filled-down.svg` → `bwi-arrow-filled-down`

### Naming Guidelines

- Use **descriptive names** that indicate what the icon represents
- Use **kebab-case** (lowercase with hyphens)
- Add suffixes for variants:
  - `-filled` for filled/solid versions (e.g., `star-filled`)
  - `-alt` for alternative styles (e.g., `edit-alt`)
  - `-1`, `-2` for numbered variants (e.g., `settings-1`)

### Common Patterns

| Category        | Example Names                      | Usage                    |
| --------------- | ---------------------------------- | ------------------------ |
| Actions         | `add`, `edit`, `delete`, `copy`    | User actions             |
| Status          | `help`, `info`, `warning`, `error` | Status indicators        |
| Navigation      | `angle-up`, `arrow-filled-down`    | Directional navigation   |
| Objects         | `login`, `note`, `collection`      | Bitwarden objects        |
| Filled variants | `star-filled`, `lock-filled`       | Filled state for toggles |

## Troubleshooting

### Icon not appearing after build

- Verify the SVG file is in `libs/assets/src/material-icons/`
- Ensure the filename matches the icon name you're using (e.g., `help.svg` for `bwi-help`)
- Run `npm run icons:build` again
- Clear browser cache

### Wrong icon displaying

- Check that your SVG filename is correct
- Ensure you're using the correct `bwi-` class name in your component
- Verify the icon was added to `BITWARDEN_ICONS` array in `icon.ts`

### Font not updating

Delete the generated font files and rebuild:

```bash
rm libs/angular/src/scss/bwicons/fonts/bwi-font.*
npm run icons:build
```

### TypeScript errors

- Ensure you added the icon to `libs/components/src/shared/icon.ts`
- Check that the icon name format matches: `bwi-icon-name`

## Advanced Usage

### Creating Icon Variants

If you need multiple CSS classes for the same icon (like `collection` and `collection-shared`), add
to `FIGMA_VARIANTS` in `scripts/material-icons/migration-map.ts`:

```typescript
export const FIGMA_VARIANTS: Record<string, string[]> = {
  // Single SVG generates two CSS classes
  collection: ["collection-shared"],
};
```

This creates both `.bwi-collection` and `.bwi-collection-shared` using the same glyph from
`collection.svg`.

## Best Practices

1. **Consistent Naming**: Use descriptive Figma names that match design system conventions
2. **Descriptive Names**: Choose names that indicate purpose (e.g., `help`, `add`, `warning`)
3. **Documentation**: Always add usage guidelines to Storybook
4. **Accessibility**: Include proper ARIA labels when using icons without text
5. **SVG Optimization**: Export from Figma with strokes outlined and optimized settings
6. **Testing**: Verify icons in Storybook before committing
7. **Alphabetical Order**: Add new icons alphabetically to `BITWARDEN_ICONS` array
