## Overview

`bit-file-upload` is the inline single-file picker. It renders a "Choose File" pill alongside a
filename readout — minimal footprint, suitable for fields that sit inside a tight form layout.

For drag-and-drop or multi-file selection, use the dedicated
[`bit-file-dropzone`](/?path=/docs/component-library-form-file-dropzone--docs) component instead.

```html
<bit-file-upload accept=".png,.jpg,.gif,.svg" formControlName="file">
  <bit-label>Upload file</bit-label>
  <bit-hint>SVG, PNG, JPG or GIF (MAX. 800x400px)</bit-hint>
</bit-file-upload>
```

The component is internally form-field-shaped — it wraps `<bit-form-field>` and projects its visual
into the prefix slot. As a result, the label, required indicator, error message, and hint inherit
`bit-form-field`'s plumbing automatically.

Set `disabled` (or bind `[disabled]="true"`) to prevent interaction. Use this when the surrounding

### Long filenames

Filenames that exceed the available width are truncated in the middle so both the start and file
extension remain visible. The first ~15% and last ~20% of the name are always shown, while the
middle ~65% is clipped with an ellipsis.

## Guidelines

### When to use this vs `bit-file-dropzone`

Use **`bit-file-upload`** for single-file fields where space is limited — for example, an avatar
upload or an inline attachment field. The picker is keyboard-accessible and uses the same form-field
shape as the rest of the component library.

Use [**`bit-file-dropzone`**](/?path=/docs/component-library-form-file-dropzone--docs) when the user
may select multiple files, when you want to encourage drag-and-drop as a primary interaction, or
when the field deserves visual prominence on the page.

### `accept` — file type hint

Pass a comma-separated list of MIME types and/or file extensions to guide the browser's file picker.

| Example value            | Allows                 |
| ------------------------ | ---------------------- |
| `"image/*"`              | Any image              |
| `".pdf,.doc,.docx"`      | PDF and Word documents |
| `"audio/mpeg,audio/wav"` | Specific audio formats |
| `"image/*,.pdf"`         | Images and PDFs        |

> **Important:** `accept` is a browser hint only — it filters the file picker but does not prevent
> users from selecting other file types. Always validate file types on the server before processing.

See
[MDN: accept attribute](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Attributes/accept)
for a full list of valid specifiers.

### Binding selected files

Bind via Angular reactive forms. The form value is always `File[]` (empty array means no file
selected); consumers read `value[0]`.

```html
<bit-file-upload accept="application/pdf" formControlName="file">
  <bit-label>Upload file</bit-label>
</bit-file-upload>
```

```ts
form = new FormGroup({
  file: new FormControl<File[]>([], { nonNullable: true }),
});

onSubmit() {
  const file = this.form.controls.file.value[0];
  // ...
}
```

You can pre-populate the field by calling `setValue([file])` on the control before the component
renders.

### Error state

The error border and message render automatically when the bound `FormControl` is `invalid` and
`touched`. Use Angular validators (or `setErrors`) on the control to drive the state.

```ts
form = new FormGroup({
  file: new FormControl<File[]>([], {
    nonNullable: true,
    validators: [
      Validators.required,
      (control) => {
        const file = control.value[0];
        return file && file.size > 30 * 1024 * 1024
          ? { tooLarge: { message: "File exceeds the 30 MB limit." } }
          : null;
      },
    ],
  }),
});
```

The component reads the first error from `control.errors` and renders it through `<bit-error>`,
matching the rest of the Bitwarden form-field family.

## Accessibility

- The selected filename is announced via an `aria-live="polite"` region so screen readers notify the
  user after selection without interrupting the current reading position.
- Always provide a meaningful `<bit-label>` — it serves as the accessible label for the picker
  button via `bit-form-field`'s `<label [for]>` association.
