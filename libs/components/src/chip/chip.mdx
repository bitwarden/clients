## Overview

Chips are compact, interactive UI elements used to represent options, values, or actions in a
lightweight, scannable format. They provide a consistent way to display and interact with discrete
pieces of information across the Bitwarden interface.

## Using chips

Use chips when information or controls need to be:

- Immediately recognizable
- Quickly actionable
- Easily grouped or compared
- Secondary to the primary content or actions on the screen

## Chip Types

The component library provides three types of chips, each designed for specific use cases:

### Chip (Base Component)

The base `bit-chip` component represents user-provided values, entities, or selections that can be
reviewed, edited, or removed. Use chips when users are entering or managing multiple discrete items,
such as tags, users, or permissions.

<Canvas of={stories.Default} />

**Use the base chip when:**

- Displaying user-entered tags or labels
- Showing selected items that can be dismissed
- Representing discrete entities with remove functionality

### Action Chip

Action chips trigger a single contextual action related to the current view or task. They help users
act quickly without changing selection state or entering input. Action chips can be implemented as
buttons or links.

**Use action chips when:**

- Providing quick access to contextual actions
- Offering lightweight interactive elements that don't change state
- Creating actions that complement but don't compete with primary buttons

See [Chip Action](?path=/docs/component-library-chip-chip-action--docs) for detailed usage,
variants, and examples.

### Filter Chip

Filter chips allow users to narrow or refine a list of items by predefined criteria. They can be
single-select or multi-select with dropdown menus, supporting both flat and nested option
structures.

**Use filter chips when:**

- Filtering lists or tables by specific criteria
- Allowing users to select from predefined options
- Supporting hierarchical filter selections

See [Chip Filter](?path=/docs/component-library-chip-chip-filter--docs) for detailed usage and
examples.

## Content Guidelines

Chip copy should be **1-2 words** for optimal scannability.

If necessary, chip text should always be truncated with ellipses, not wrapped to a second line.

## Base Chip Component (`bit-chip`)

The sections below describe the specific functionality and usage of the base chip component. For
general information about chips and other chip types, see the overview above.

### Basic Usage

The base chip component displays a label with an optional start icon and a dismiss button.

```html
<bit-chip label="Tag name" startIcon="bwi-tag" (chipDismissed)="handleRemove()"> </bit-chip>
```

```ts
@Component({
  selector: "app-my-component",
  template: `
    <div class="tw-flex tw-flex-wrap tw-gap-2">
      <bit-chip
        *ngFor="let tag of tags"
        [label]="tag.name"
        startIcon="bwi-tag"
        (chipDismissed)="removeTag(tag)"
      >
      </bit-chip>
    </div>
  `,
})
class MyComponent {
  protected tags = [
    { id: 1, name: "Work" },
    { id: 2, name: "Personal" },
    { id: 3, name: "Important" },
  ];

  protected removeTag(tag: Tag) {
    this.tags = this.tags.filter((t) => t.id !== tag.id);
  }
}
```

### Dismissing Chips

The chip component includes a built-in dismiss button (X icon) that emits a `chipDismissed` event
when clicked. Handle this event to remove the chip from your collection.

```html
<bit-chip label="Removable tag" (chipDismissed)="handleDismiss()"> </bit-chip>
```

### Sizes

Chips are available in two sizes: small and large (default).

```html
<!-- Small size -->
<bit-chip label="Small" size="small"></bit-chip>

<!-- Large size (default) -->
<bit-chip label="Large" size="large"></bit-chip>
```

### Disabled State

Chips can be disabled to prevent interaction.

```html
<bit-chip label="Disabled chip" [disabled]="true"> </bit-chip>
```

### With Icons

Add a start icon to provide additional visual context.

```html
<bit-chip label="User" startIcon="bwi-user"> </bit-chip>

<bit-chip label="Folder" startIcon="bwi-folder"> </bit-chip>
```

### Accessibility

- The chip includes proper `aria-disabled` attributes when disabled
- The dismiss button includes an internationalized aria-label for screen readers
- Ensure sufficient color contrast when using chips on different backgrounds
- Use descriptive labels that clearly identify what the chip represents
